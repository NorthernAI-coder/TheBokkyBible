#### Quick Context on dGEN1 (from 2025–2026 releases)
- dGEN1 is Freedom Factory’s “onchain everyday carry” — a hardware wallet / DeFi device running ethOS, built around **account abstraction** with private keys locked in a secure enclave (TEE).  
- It was marketed as one of the “hardest” hardware wallets: keys supposedly non-extractable even by software updates, and **no traditional mnemonic seed phrase** needed for daily use.  
- Instead, it uses a **Burner Card** (an NFC-enabled recovery/backup card) that acts as the functional equivalent of a seed phrase. Tapping the card during setup or recovery designates it as the backup mechanism and allows restoring funds to a different address if the main dGEN1 is lost/damaged.  

So when people say “extract the seed phrase from the DGEN1 phone backup card,” they’re usually talking about pulling data from that NFC Burner Card.

#### Flipper Zero Capabilities (Reality Check in 2026)
Flipper Zero excels at **NFC/RFID reading, cloning, and emulation** — especially on older or less-secured protocols (MIFARE Classic Gen1/2, NTAG, etc.). It can:
- Read and save data from many NFC cards.
- Perform attacks on weakly protected cards (nested attacks, key recovery on CRYPTO1, etc.).
- Clone or emulate cards for testing/auditing.

However:
- Modern secure enclaves and properly implemented NFC wallets (especially those using EVM account abstraction with hardware-backed keys) are designed to resist casual extraction.
- If the Burner Card stores sensitive recovery data in plaintext or a weak format, a Flipper (or any NFC reader + custom script) could potentially dump it with physical proximity.
- Many in the community use Flipper for legitimate security research or personal recovery testing, but it also highlights why storing raw seed-like data on simple NFC tags is risky.
